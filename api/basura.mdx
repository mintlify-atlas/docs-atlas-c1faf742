---
title: "Trash detection API"
description: "REST endpoints for trash/waste detection statistics and analytics."
---

**Base URL**: `http://localhost:8080`

<Note>
  The `basura-deteccion-service` uses MongoDB as its data store and does not define an explicit `server.port`. When run locally outside Docker it defaults to port `8080`.
</Note>

---

## GET /api/detections

Returns all raw trash detection records stored in the `detections` collection.

### Response

Returns a JSON array of detection documents.

<ResponseField name="_id" type="string">
  MongoDB document identifier.
</ResponseField>

<ResponseField name="id" type="integer">
  Numeric label ID corresponding to the trash category.
</ResponseField>

<ResponseField name="etiqueta" type="string">
  Human-readable label for the detected waste type.
</ResponseField>

<ResponseField name="total" type="integer">
  Running total count for this label.
</ResponseField>

<ResponseField name="conteo" type="integer">
  Detection count for this record.
</ResponseField>

<ResponseField name="dia_semana" type="string">
  Day of the week the detection occurred.
</ResponseField>

<ResponseField name="fecha_hora" type="string">
  ISO-8601 datetime of the detection.
</ResponseField>

### Example

```bash
curl http://localhost:8080/api/detections
```

## GET /api/test-count

Returns the raw document count from the `detections` MongoDB collection. Useful for verifying database connectivity.

### Response

<ResponseField name="body" type="integer">
  Number of documents in the `detections` collection.
</ResponseField>

### Example

```bash
curl http://localhost:8080/api/test-count
```

## GET /api/db-info

Returns the active MongoDB database name and a list of its collection names.

### Response

<ResponseField name="database" type="string">
  Name of the connected MongoDB database.
</ResponseField>

<ResponseField name="collections" type="array">
  List of collection name strings present in the database.
</ResponseField>

### Example

```bash
curl http://localhost:8080/api/db-info
```

---

## GET /api/ping

Runs a MongoDB ping command and returns its result. Useful for verifying the database connection is healthy.

### Response

<ResponseField name="body" type="string">
  `"MongoDB PING OK: ..."` on success, or `"MongoDB PING FAILED: ..."` with the error message on failure.
</ResponseField>

### Example

```bash
curl http://localhost:8080/api/ping
```

## GET /api/totales

Returns the total detection count aggregated per waste label.

### Response

Returns a JSON array of objects.

<ResponseField name="etiqueta" type="string">
  Waste label name.
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of detections for this label.
</ResponseField>

### Example

```bash
curl http://localhost:8080/api/totales
```

## GET /api/proporciones

Returns each waste label's proportion of total detections as a decimal fraction.

### Response

Returns a JSON array of objects.

<ResponseField name="etiqueta" type="string">
  Waste label name.
</ResponseField>

<ResponseField name="proporcion" type="number">
  Proportion of total detections for this label (0.0 – 1.0).
</ResponseField>

### Example

```bash
curl http://localhost:8080/api/proporciones
```
