The Espacios Seguros IA backend is composed of multiple Spring Boot microservices and a MySQL database, all orchestrated with Docker Compose.

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) installed and running
- [Docker Compose](https://docs.docker.com/compose/install/) v2 or higher

## Services overview

All services are connected via a shared `micro-red` bridge network.

| Service | Host port | Container port | Description |
|---|---|---|---|
| `mysql-db` | 3307 | 3306 | MySQL 8 database |
| `gestion-usuarios-service` | 8081 | 8080 | User management service |
| `alerta-congestion-service` | 8082 | 8080 | Congestion alert service |
| `conteo-zona-service` | 8083 | 8080 | Zone counting service |
| `personas_detectadas` | 8084 | 8080 | Detected persons service |
| `agresiones-service` | 8085 | 8080 | Aggression events service |

## Deployment steps

<Steps>
  <Step title="Navigate to the microservices directory">
    ```bash
    cd microservices/
    ```
  </Step>

  <Step title="Start all services in detached mode">
    ```bash
    docker-compose up -d
    ```

    Docker will build images for all Spring Boot services and pull the MySQL image on first run. This may take a few minutes.
  </Step>

  <Step title="Verify all services are running">
    ```bash
    docker-compose ps
    ```

    All services should show a `running` status. If any service has exited, inspect its logs:

    ```bash
    docker-compose logs <service-name>
    ```
  </Step>
</Steps>

## Environment variables

Each Spring Boot microservice is configured with the following environment variables to connect to the MySQL database:

| Variable | Value | Description |
|---|---|---|
| `SPRING_DATASOURCE_URL` | `jdbc:mysql://mysql-db:3306/servicio` | JDBC connection string (uses the internal Docker hostname) |
| `SPRING_DATASOURCE_USERNAME` | `root` | Database username |
| `SPRING_DATASOURCE_PASSWORD` | `mosca_311` | Database password |

The MySQL database is accessible from the **host machine** at `localhost:3307` (mapped from port 3306 inside the container).

## docker-compose.yml reference

```yaml
services:
  mysql-db:
    image: mysql:8
    container_name: mysql-db
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: mosca_311
      MYSQL_DATABASE: servicio
      MYSQL_PASSWORD: mosca_311
    ports:
      - "3307:3306"
    volumes:
      - mysql-data:/var/lib/mysql
    networks:
      - micro-red

  gestion-usuarios-service:
    build:
      context: ./gestion-usuarios-service
    container_name: gestion-usuarios-service
    depends_on:
      - mysql-db
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql-db:3306/servicio

      - "8081:8080"
    networks:
      - micro-red
```

<Note>
  The AI Python detection scripts (YOLO/DeepSORT) are **not** included in the Docker Compose stack and must be run separately. When running the Python scripts alongside the microservices stack, configure them to connect to MySQL at `localhost:3307` (or the appropriate host if running on a remote machine).
</Note>

## Stopping the stack

```bash
# Stop all services (keeps containers and volumes)
docker-compose stop

# Stop and remove containers
docker-compose down

# Stop and remove containers + volumes (deletes all stored data)
docker-compose down -v
```
